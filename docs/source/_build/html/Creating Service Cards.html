Creating an Application¶

Creating an application requires entering several pieces of information. Every application needs a name, the type of application, the organization the application belongs to and to specify what policy should be displayed to citizen’s when they connect to a service for the first time. After creating the application, the administrator will be provided with two keys – the Client Id and the Client Secret. These two keys are required when the application is making calls to the CitizenOne API. This is the only time the Client Secret is visible to an administrator so the keys should be recorded in a secure location.

Creating a Service Card¶

Service Cards allows administrators to view all service cards that have been created in the system.

The Details tab allows an administrator to view the service card title, service type, service description and status of pilot mode.
